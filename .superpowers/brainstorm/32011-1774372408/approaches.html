Policy Creator: Design Approaches

Three different architectures for the visual policy builder

A

Single-Page Tabbed Builder

Tabs for Guardrails, Routing, PII sections. Condition builder at top. Actions configured per-tab. Compact, all-in-one view.

Pros

  • Everything visible at once
  • Fast navigation between sections
  • Simpler implementation

Cons

  • Can feel overwhelming with many actions
  • Less guided experience
B

Wizard-Based Builder

Step-by-step flow: Basic Info → Select Category → Build Conditions → Configure Actions → Review. Guided experience.

Pros

  • Great for non-technical users
  • Guided, less overwhelming
  • Clear progress indication

Cons

  • Slower for power users
  • More clicks to complete
  • Harder to edit existing policies
C

Hybrid Builder with Mode Toggle (Recommended)

Visual mode by default with tabbed sections. Toggle to JSON mode for developers. Best of both worlds.

Pros

  • Serves both user types
  • Visual for non-technical, JSON for developers
  • Flexible editing patterns

Cons

  • More complex implementation
  • Need to keep both modes in sync

Recommendation: Hybrid Builder (Approach C)

Since we need to serve both non-technical admins and developers, the hybrid approach with a mode toggle is ideal. The visual builder provides guided, dropdown-based editing for conditions and actions, while the JSON toggle lets developers work efficiently with raw policy definitions.
